Privacy Policy

Last updated: June 2026

AI News Short Factory ("the Tool", "we", "our") is a local, single-creator workflow tool. This Privacy Policy explains what data the Tool processes, how it is stored, and your rights in relation to that data.

Because the Tool runs entirely on your own hardware, the vast majority of data never leaves your machine. This policy is provided in support of third-party platform integrations (including TikTok) that require a published privacy policy.

1. Who this applies to

AI News Short Factory is designed for use by a single authorized creator — the person who installs and operates the Tool. There are no end-user accounts, no public-facing sign-up flows, and no multi-tenant data processing.

2. Data collected and how it is used

The Tool processes the following categories of data:

2a. TikTok OAuth tokens

When you authorize the Tool to deposit draft videos to your TikTok account, TikTok issues an OAuth access token and refresh token. These tokens are:

  • Stored locally on your machine in a configuration file accessible only to you.
  • Used solely to call the TikTok Content Posting API on your behalf.
  • Never transmitted to any server, cloud service, or third party controlled by the Tool.
  • Subject to TikTok's own token expiry and revocation mechanisms.
You can revoke access at any time via TikTok's Settings → Security → Authorized apps. Revocation immediately invalidates all tokens issued to this Tool.

2b. Generated video metadata

The Tool generates and stores the following locally as part of normal pipeline operation:

  • News story sources (URLs, headlines, publication dates) used for script research.
  • AI-generated script text and fact-check results.
  • Rendered video files and associated captions.
  • Pipeline run logs and stage reports (timing, error messages, scoring data).

This data is stored on your local machine in the Tool's working directory. It is not uploaded to any cloud service or third party, except that video files are uploaded to TikTok as private drafts when you trigger an upload.

2c. Local storage

All persistent state — including configuration, pipeline history, stage outputs, SQLite databases, and logs — is stored on-disk in the Tool's local working directory. No cloud database, analytics service, or telemetry endpoint is used.

2d. Third-party AI services

The Tool sends news story text and script content to third-party large language model APIs (such as OpenAI or Anthropic) to generate scripts and fact-check results. No personal identifiable information about the creator or any individual is deliberately included in these requests. You should review the privacy policies of any AI API providers you configure.

3. Data we do not collect

  • We do not collect or store TikTok viewer data, analytics, comments, or follower information.
  • We do not operate any server that receives data from your machine.
  • We do not use tracking pixels, cookies, or browser fingerprinting.
  • We do not collect personal information from third parties or data brokers.

4. Sale of data

We do not sell, trade, rent, or otherwise transfer your data — or any data generated by the Tool — to any third party, for any purpose, at any time.

5. Data retention

Data is retained on your local machine for as long as you keep it. You can delete any or all pipeline outputs, logs, and configuration files at any time by removing the relevant files from the Tool's working directory.

TikTok OAuth tokens remain valid until you revoke them inside TikTok, or until they expire per TikTok's token lifecycle policy.

6. How to request deletion

Because all data is stored locally on your machine, you can delete it at any time:

  • Pipeline data and logs: Delete files from the Tool's working directory on your machine.
  • TikTok OAuth tokens: Revoke access via TikTok → Settings → Security → Authorized apps, then delete the local token file from the Tool's configuration directory.
  • Uploaded drafts: Delete video drafts directly inside TikTok.

For any questions about data deletion, contact us at the address below.

7. Security

Token files and configuration are stored with filesystem permissions restricted to your local user account. We recommend keeping your machine's operating system and storage encrypted. The Tool has no network-accessible components — it does not run a public server.

8. Children's privacy

The Tool is intended for use by adult content creators. It is not directed at children under 13 (or the applicable age of digital consent in your jurisdiction), and we do not knowingly process any data relating to minors.

9. Changes to this policy

If we make material changes to this policy, we will update the "Last updated" date above and note changes in the project changelog. Continued use of the Tool after changes constitutes acceptance of the revised policy.

10. Contact

Questions about this Privacy Policy can be directed to the project maintainer via the project's GitHub repository issues page or by email at the address listed in the project README.
